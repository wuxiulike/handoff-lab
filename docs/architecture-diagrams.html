工作流

sequenceDiagram participant U as User participant S as Handoff Lab Server participant C as Codex-style Planner/Reviewer participant R as Reasonix-compatible Worker participant V as QA Viewer U->>S: 提交任务和工作目录 S->>V: 推送任务开始事件 S->>C: 请求规划和验收标准 C-->>S: 返回任务包、计划、acceptance S->>R: 发送实现任务包 R->>R: 修改代码、运行命令、生成证据 R-->>S: 返回 build report / test log / artifact paths S->>C: 提交证据包进行 review C-->>S: APPROVED 或 CHANGES_REQUESTED alt 需要修复 S->>R: 发送 next_fix 精准修复指令 R-->>S: 重新提交证据 S->>C: 复审 else 通过 S->>V: 展示完成状态和证据摘要 S-->>U: 等待用户最终确认 end
